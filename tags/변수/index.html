Variables (변수)

Variables

1) 변수의 정의

  • 변수란 하나의 값을 저장하기 위해 확보한 메모리 공간 자체와 그 공간을 식별하기 위해 붙인 이름입니다.

2) (Why?) 왜 변수란 것이 필요한가?

변수는 왜 필요한 것일까요?

다음과 같은 식을 코드에서 실행한다고 생각해봅시다.

자바스크립트 엔진이 위의 식을 해석하면 10과 20은 컴퓨터에 메모리에 담기고, + 연산자는 cpu에서 실행되게 됩니다. 이 때 메모리에 10과 20은 그리고 연산 결과는 2진수의 형태로 메모리 셀에 저장되게 됩니다.

이렇게 30을 연산했다는 것은 그 값을 사용해 무언가를 하겠다는 의도가 있을 것입니다. 한번 사용이 아닌 여러번 사용하기 위해서는 메모리에 직접적인 접근이 필요합니다.

2.1) 자바스크립트는 메모리에 직접적인 접근을 허용하지 않는다.

그러나, 메모리에 직접 접근하는 것은 치명적인 오류를 발생시킬 수 있습니다. 만약 실수로 운영체제가 사용하고 있는 값을 변경하게 되면 시스템이 멈출 수도 있습니다. 따라서 자바스크립트는 메모리에 직접적인 접근을 허용하지 않습니다.

2.2) 메모리에 직접적인 접근이 가능하다면?

메모리에 대한 접근을 허용하게 되더라도 문제가 있습니다. 값이 저장될 메모리 주소는 코드가 실행될 때마다 메모리 상황에 따라 임의로 결정됩니다. 따라서 동일한 코드를 실행해도 실행마다 값이 저장될 메모리 주소가 변경되고, 이전 값이 저장된 주소를 알수도 없고, 알려주지도 않기 때문에 메모리에 직접 접근하는 것은 올바른 방법이 아닙니다.

$ 결론

메모리에 접근할 수 있도록 하는 변수를 이용하여 값에 안전하게 접근하고 사용할 수 있습니다.

3) 식별자

식별자는 어떠한 값을 구별해서 식별할 수 있는 고유한 이름을 말합니다. 앞서 말한 변수 또한 값에 접근, 즉 식별할 수 있으므로 식별자 입니다. 이 식별자는 값 자체가 아닌 메모리 주소를 기억하고 있습니다. 즉 식별자인 변수 이름에 저장된 메모리 주소를 통해 변수 값이 저장되어있는 메모리에 접근하는 것입니다.

3.1) 식별자 네이밍규칙

이러한 식별자들은 다음과 같은 네이밍 규칙을 지켜야 합니다.

  1. 식별자는 특수문자를 제외한 문자, 숫자, 언더스코어(_), 달러기호($)를 포함할 수 있다.
  2. 식별자는 특수문자를 제외한 문자, 언더스코어(_). 달러기호($)로 시작해야한다. 숫자로 시작은 불가능하다.
  3. 예약어는 식별자로 사용할 수 없다.

4) 변수 선언

변수를 사용하기 위해서는 변수 선언을 해야됩니다.식별자는 선언에 의해 자바스크립트 엔진에 식별자의 존재를 알립니다. 식별자가 참조할 메모리 공간을 확보하고 (이때 확보는 “나는 이 식별자의 값으로 쓰일 거야, 다른 애들은 건드릴 수 없어!”라고 알리는 것), 그 식별자를 실행 컨텍스트에 등록하게 됩니다.

변수 선언은 var, let, const 키워드를 사용합니다. es6에 추가된 let과 const는 나중에 자세히 다룰테니 지금 설명은 var를 기준으로 하겠습니다.

이 때 공간을 확보한다고 했으니 그 메모리가 비어있다고 생각할 수 있는데 사실 메모리는 비어있는 것이 아닌 이전에 다른 어플리케이션에서 사용하고 참조가 끊어진 값(쓰레기값)이 들어있으며 자바스크립트 엔진은 암묵적으로 undefined를 할당하여 초기화하게 됩니다.

4.1) (Why?) 왜 undefined로 초기화하는 과정을 거칠까?

초기화 과정을 거치지 않는다면 이전에 썼던 쓰레기 값이 들어있는 메모리셀을 참조하게 되기 때문에 할당 직전에 값을 참조하게 될 경우 원하지 않는 동작이 일어날 수 있습니다.
따라서 자바스크립트에서는 암묵적으로 undefined를 할당하는 것으로 초기화과정을 거쳐 이러한 위험을 미연에 방지합니다.

4.2) 변수 선언의 시점

그러면 변수는 언제 선언될까요? 다음 코드를 먼저 보겠습니다.

만약 코드가 실행되는 ‘런타임’에 선언된다면 선언문 위에 있는 console.log(hoi)는 아직 선언 이전이기 때문에 ‘식별자를 통해 참조하려 했으나 식별자를 찾을 수가 없어!’라는 뜻의 Reference 에러를 내야합니다. 그러나 코드가 정상 동작하는 것을 확인할 수 있었습니다. 이것을 통해 저희는 두가지를 알 수 있습니다.

  1. 변수의 선언은 코드가 실행되는 런타임이 아닌 그 이전 단계에서 미리 실행 된다는 것
  2. var로 선언한 코드는 런타임 이전에 선언뿐만 아니라 undefined로 초기화까지 일어난다는 것

$ 실제 동작

자바스크립트 엔진은 코드를 읽기 전에 가장 먼저 (런타임 이전에) 전체 코드에서 선언문을 전부 가져와 실행컨텍스트에 등록을 하게 됩니다. var의 경우에는 선언과 동시에 초기화가 일어나게 됩니다. 이렇게 실제 코드가 존재하는 곳과 달리 런타임이전에 선언이 되어서 “마치 선언문이 코드의 최상단으로 끌어올려진 것처럼 동작하는 것”을 호이스팅이라고 합니다.

자바스크립트에서는 변수 선언뿐 아니라 var, let, const에 더하여 function, class 키워드를 사용해서 선언하는 모든 식별자는 호이스팅 됩니다.

5) 변수 할당

위 코드는 하나의 문처럼 보이지만 사실 두개의 문의 단축 표현입니다. 즉 자바스크립트는 변수 선언과 값의 할당을 2개의 문으로 나누어 각각 실행합니다. 따라서 단축 표현시에도 undefined가 먼저 할당되어 초기화 된 이후 100이라는 값이 할당 되는 것에는 변함이 없습니다.

5.1) 할당 시점

var 키워드로 변수를 선언하게 되면 소스코드 평가과정에서 변수를 실행컨텍스트에 등록하고, 초기화까지 한다고 했습니다. 그 이후 실제 코드가 읽히는 런타임에서 값을 할당하게 됩니다.

앞에서 사용한 예시를 통해 코드를 읽은 이후에 20이라는 값을 할당했고, 그 이전에는 선언과 초기화 과정만을 거쳤음을 알 수 있습니다.

5.2) 변수 값과 메모리

변수에 값을 할당할 때, undefined가 저장되어있는 메모리 공간을 지우고 그 메모리 공간에 할당 값 80을 다시 넣어주는 것이 아니라는 것을 명심해야 합니다. 값을 할당할 때에는 변수가 참조할 메모리 영역을 새로 확보하고 그 메모리 영역을 변수가 참조하게 됩니다.

5.3) 재할당

재할당도 마찬가지 입니다. hoi라는 변수가 가리키고 있는 20의 값을 가진 메모리에 접근하여 값을 변경시키는 것이 아니라 새로운 메모리 공간을 확보하여 값을 저장합니다.

$ 정리

변수가 var, let, const 라는 것이 있고 값을 저장하기 위해 사용한다라는 당연한 이야기를 메모리의 관점에서 살펴보고, 실제 변수가 선언되고 할당되는 시점에 대해서 생각해봤습니다. 변수 호이스팅의 개념과 메모리 안에 있는 값 자체를 바꾸는 것이 아니라는 점이 포인트였던 단원이였습니다.
